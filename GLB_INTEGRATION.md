# GLB 模型集成说明

## ✅ 已完成的工作

### 1. 文件结构
```
ha-homeui/
├── public/
│   └── models/
│       └── huxingtu.glb (27.01 MB)  # 户型图 3D 模型
├── src/
│   ├── loaders/
│   │   └── ModelLoader.js           # GLTFLoader 封装
│   ├── scene/
│   │   └── SceneManager.js          # 已更新：加载 GLB 模型
│   └── main.js                      # 已更新：支持异步初始化
```

### 2. 新增功能

#### ModelLoader.js
- 封装了 Three.js 的 GLTFLoader
- 支持模型缓存
- 自动配置阴影
- 支持缩放、位置、旋转设置
- 加载进度回调

#### SceneManager.js
- 新增 `loadHouseModel()` 方法加载 GLB 模型
- 自动计算模型边界和尺寸
- 失败时回退到程序化生成房间
- 输出详细的加载日志

#### main.js
- `init()` 方法改为异步
- 等待场景加载完成后再隐藏 loading

### 3. 使用方式

启动开发服务器：
```bash
npm run dev
```

访问 http://localhost:3000，控制台会显示：

### 4. 特性

✅ **自动阴影** - 模型中的所有 Mesh 自动启用阴影投射和接收  
✅ **进度显示** - 控制台实时显示加载进度  
✅ **错误处理** - 加载失败时自动回退到程序化生成的房间  
✅ **模型缓存** - 重复加载同一模型时使用缓存  
✅ **边界计算** - 自动计算模型尺寸，便于调整相机位置  

### 5. 调整建议

如果模型显示不正确，可以调整以下参数：

**缩放**
```javascript
scale: 0.1  // 模型太大时缩小
```

**位置**
```javascript
position: { x: 0, y: 0, z: 0 }  // 调整模型位置
```

**旋转**
```javascript
rotation: { x: 0, y: Math.PI, z: 0 }  // 旋转 180 度
```

**相机位置** (在 main.js 中)
```javascript
this.camera.position.set(8, 8, 12)  // 根据模型尺寸调整
```

## 📝 注意事项

1. GLB 文件较大（27MB），首次加载可能需要几秒钟
2. 模型已启用阴影，确保场景中有光源
3. 如果模型不可见，检查控制台的尺寸信息，可能需要调整相机位置
4. 原来的程序化生成房间已被注释，如需启用可取消注释 `this.buildHouse()`
