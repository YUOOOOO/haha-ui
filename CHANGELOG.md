# HA Home UI - 更新日志

## v1.2 - 设置页面与配置管理 (2026-06-01)

### 🎉 新增功能

#### 1. 设置页面
- ✅ 创建了独立的设置页面 (`settings.html`)
- ✅ 现代化的 UI 设计，支持响应式布局
- ✅ 在主页面底部添加了设置按钮
- ✅ 设置页面在新窗口中打开

#### 2. 天气服务配置
- ✅ 支持三种天气服务提供商：
  - **wttr.in**（免费，无需 API Key）
  - **和风天气**（推荐国内用户）
  - **OpenWeatherMap**（国际服务）
- ✅ 可视化的 API Key 输入（支持显示/隐藏）
- ✅ 自动定位或手动指定城市
- ✅ 每个提供商都有详细的使用说明和注册链接

#### 3. Home Assistant 配置
- ✅ 自定义服务器地址
- ✅ 自定义访问令牌
- ✅ 测试连接功能（验证配置是否正确）
- ✅ 显示 Home Assistant 版本号

#### 4. 显示设置
- ✅ 显示/隐藏天气信息
- ✅ 显示/隐藏日期
- ✅ 时间格式切换（12小时制/24小时制）
- ✅ 温度单位切换（摄氏度/华氏度）

#### 5. 设置管理
- ✅ 所有设置保存在浏览器 localStorage
- ✅ 支持恢复默认设置
- ✅ 设置立即生效（刷新页面后）
- ✅ 支持导出/导入设置（高级功能）


### 💡 使用方法

1. **打开设置页面**
   - 点击主页面底部的 **⚙️ 设置** 按钮

2. **配置天气服务**
   - 选择天气服务提供商
   - 如需要，输入 API Key
   - 设置位置（auto 或城市名称）

3. **配置 Home Assistant**
   - 输入服务器地址（如 `http://192.168.1.100:8123`）
   - 输入长期访问令牌
   - 点击 **测试连接** 验证配置

4. **配置显示选项**
   - 选择是否显示天气和日期
   - 选择时间格式（12/24小时制）
   - 选择温度单位（°C/°F）

5. **保存设置**
   - 点击 **💾 保存设置** 按钮
   - 刷新主页面使设置生效

### 🎨 界面改进

- 设置页面采用渐变背景，与主页面风格一致
- 表单元素使用圆角设计，现代化风格
- 按钮有悬停动画效果
- 响应式布局，支持移动设备
- 消息提示（成功/错误）自动消失

### 🔒 安全性

- API Key 和访问令牌仅保存在本地浏览器
- 支持密码输入框（可显示/隐藏）
- 不会上传任何敏感信息到服务器

### 📚 文档

- **SETTINGS_GUIDE.md** - 详细的设置页面使用说明
- **WEATHER_README.md** - 天气服务配置说明
- 包含常见问题解答和故障排查

## v1.1 - 天气服务模块 (2026-06-01)

### 🎉 新增功能

#### 1. 独立的天气服务模块
- ✅ 创建了 `WeatherService` 类
- ✅ 支持多个公开天气 API
- ✅ 统一的数据格式
- ✅ 自动缓存（30分钟）

#### 2. 支持的天气服务
- **wttr.in**（默认）
  - 免费，无需注册
  - 无需 API Key
  
- **和风天气**
  - 国内服务，速度快
  - 需要 API Key
  - 免费额度：每天 1000 次
  
- **OpenWeatherMap**
  - 国际服务
  - 需要 API Key
  - 免费额度：每分钟 60 次

#### 3. 功能特性
- ✅ 自动定位（基于 IP）
- ✅ 手动指定城市
- ✅ 降级处理（API 失败时使用缓存）
- ✅ 天气代码映射（统一格式）

## 升级指南

### 从 v1.0 升级到 v1.2

1. **拉取最新代码**
   ```bash
   git pull
   ```

2. **无需安装新依赖**
   - 所有新功能使用原生 JavaScript
   - 无需运行 `npm install`

3. **首次使用**
   - 打开主页面
   - 点击 **⚙️ 设置** 按钮
   - 配置天气服务和 Home Assistant
   - 保存设置并刷新页面

4. **迁移旧配置**
   - 旧的配置文件仍然有效
   - 设置页面会使用旧配置作为默认值
   - 可以在设置页面中修改并保存
